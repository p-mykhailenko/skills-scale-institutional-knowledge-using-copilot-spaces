# Stakeholder Communication Checklist

This checklist ensures consistent, effective communication with project stakeholders throughout the project lifecycle. Use this template to maintain transparency and alignment.

_Reference: This template addresses communication gaps identified in [issue #4](https://github.com/p-mykhailenko/skills-scale-institutional-knowledge-using-copilot-spaces/issues/4)._

---

## Project Initiation Communication

- [ ] Identify all stakeholders (internal and external)
- [ ] Document stakeholder communication preferences (email, meetings, Slack, etc.)
- [ ] Establish communication frequency for each stakeholder group
- [ ] Schedule project kickoff meeting with all key stakeholders
- [ ] Share project charter and objectives with stakeholders
- [ ] Confirm stakeholder availability for key milestones and reviews
- [ ] Set up communication channels (Slack channels, email lists, meeting series)
- [ ] Document escalation paths and emergency contact procedures

## Ongoing Project Communication

- [ ] Send weekly status updates to stakeholders (using agreed format)
- [ ] Update project dashboard/board for stakeholder visibility
- [ ] Conduct bi-weekly or monthly stakeholder review meetings
- [ ] Communicate risks and issues as they arise (within 24 hours for critical items)
- [ ] Share decisions made and rationale with affected stakeholders
- [ ] Document and distribute meeting minutes within 24 hours
- [ ] Maintain an updated FAQ or knowledge base for common questions
- [ ] Celebrate milestones and successes with stakeholders

## Milestone & Release Communication

- [ ] Send advance notice (1-2 weeks) for upcoming milestones or releases
- [ ] Provide detailed release notes and feature documentation
- [ ] Conduct demo or walkthrough sessions for major releases
- [ ] Gather and document stakeholder feedback
- [ ] Communicate any changes to scope, timeline, or deliverables
- [ ] Share post-release metrics and success indicators
- [ ] Update stakeholders on any issues discovered and remediation plans

## Change Management Communication

- [ ] Communicate proposed changes with clear impact assessment
- [ ] Allow adequate review time for stakeholder feedback (minimum 3-5 business days)
- [ ] Document stakeholder concerns and how they were addressed
- [ ] Notify stakeholders of approved changes and implementation timeline
- [ ] Provide training or documentation for significant changes
- [ ] Follow up with stakeholders post-implementation for feedback

## Crisis & Escalation Communication

- [ ] Activate emergency communication plan for critical issues
- [ ] Notify Project Sponsor and key stakeholders immediately
- [ ] Provide regular updates (hourly or daily) until resolution
- [ ] Document incident timeline and communication log
- [ ] Conduct post-incident review with stakeholders
- [ ] Share lessons learned and preventive measures

## Project Closure Communication

- [ ] Schedule project closure meeting with stakeholders
- [ ] Share final project report including outcomes and metrics
- [ ] Conduct lessons learned session and document feedback
- [ ] Communicate handoff procedures for ongoing support
- [ ] Thank stakeholders for their participation and support
- [ ] Archive project documentation in accessible location
- [ ] Share success stories and best practices with broader organization

---

## Communication Best Practices

**Clarity**: Use clear, jargon-free language appropriate for your audience.

**Consistency**: Use standardized formats and templates for regular communications.

**Timeliness**: Communicate proactively; don't wait for stakeholders to ask for updates.

**Transparency**: Share both good and bad news openly; address concerns directly.

**Two-way**: Encourage stakeholder feedback and questions; communication is a dialogue.

**Documentation**: Keep records of key communications and decisions for future reference.

**Accessibility**: Ensure all stakeholders can access project information easily.
